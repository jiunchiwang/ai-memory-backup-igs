## Telegram 排版美化方案

### 現狀分析

| 環節 | 現在做法 |
|------|---------|
| Streaming 中途 | `editMessageText(content)` — **無 parse_mode**（純文字） |

| 指令回覆 (`/help` 等) | `parse_mode: "Markdown"` |

**問題**：主 agent 的回覆全是純文字，agent 輸出的 `**bold**`、`` `code` ``、```code block``` 全部原樣顯示，沒有渲染。

---

### 設計方案：分層 HTML 格式化

選 HTML 而非 MarkdownV2，因為：
- Agent 輸出常含 `_`、`*`、`[`、`]` 等字元，MarkdownV2 跳脫規則極嚴格會導致大量 400 error
- HTML 只需 escape `<>&`，可控性高
- 可以精準 fallback（parse error 時去掉 tags 變純文字）

### 架構

```
agent raw output
      │
      ▼
observerTransformer.transform()   ← 現有，剝 token
      │
      ▼ text (stripped)
      │
┌─────┴─────┐
│ formatHtml()  │  ← 新增，Markdown→HTML 轉換
└─────┬─────┘
      │
      ▼ { html, fallbackText }
      │
editMessageText(html, { parse_mode: "HTML" })
  catch → editMessageText(fallbackText)   ← 0-risk fallback
```

### `formatHtml()` 轉換規則

只做 agent 常用且安全的子集：

| Agent 寫法 | 轉為 HTML | 優先級 |
|-----------|-----------|--------|
| ` ```lang\n...\n``` ` | `<pre><code class="language-lang">...</code></pre>` | P0（最常見） |
| `` `inline` `` | `<code>inline</code>` | P0 |
| `**bold**` 或 `__bold__` | `<b>bold</b>` | P1 |
| `*italic*` 或 `_italic_` | `<i>italic</i>` | P1 |
| `~~strike~~` | `<s>strike</s>` | P2（少用） |
| `[text](url)` | `<a href="url">text</a>` | P2 |
| `> quote` | `<blockquote>quote</blockquote>` | P2 |

**不做的事**（降低風險）：
- 不處理巢狀格式（`**bold _italic_**` 不嘗試正確巢狀）
- 不處理表格、標題（等 Rich Messages 成熟再走那條路）
- code block 內不做任何二次轉換

### Streaming 策略

| 階段 | parse_mode | 理由 |
|------|-----------|------|
| Streaming 中途 edit | `"HTML"` | 即時美化；code block 未閉合時整段 escape 當純文字顯示 |
| 最終 edit | `"HTML"` | 完整轉換 |
| 400 fallback | 去掉 parse_mode | 100% 不會丟訊息 |

### 未閉合 code block 處理

Streaming 時 agent 可能寫到一半：
```
Here is the code:
```python
def foo():
    pass
```
→ 偵測到 ` ``` ` 開了但沒關 → 該 block 之後的文字全部 HTML escape 不轉換，避免 parse error。

### 實作範圍（3 個檔案）

1. **新增 `src/format-html.ts`**
   - `markdownToHtml(text: string): { html: string; plain: string }`
   - 輸入是 token-stripped 的純文字，輸出 HTML + fallback 純文字
   - ~100 行，regex-based 逐行處理

2. **修改 `src/telegram-ui.ts` — `renderReply()`**
   - 回傳 `{ content: string; parseMode: "HTML" | undefined }`
   - streaming 時用 `formatHtml`，失敗 fallback 空 parseMode

3. **修改 `src/run-prompt.ts`**
   - `editMessageText` 加 `{ parse_mode: "HTML" }` + catch fallback
   - 最終 chunk 送出也加

### 風險控制

| 風險 | 對策 |
|------|------|
| HTML parse error 導致 400 | 每次 editMessageText 的 catch 裡 fallback 到無 parse_mode |
| Agent 輸出 `<script>` 等 HTML | 所有非轉換的 `<` `>` `&` 都先 escape |
| 效能（每次 edit 都要跑轉換） | markdownToHtml 是純 regex，<1ms |
| Streaming 半開 tag | 未閉合 code block → 跳過轉換那段 |
