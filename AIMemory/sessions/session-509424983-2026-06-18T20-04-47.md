### 關鍵架構特色（最新）

**1. 狀態機（19 state，GAMEVIEW_STATE）**

```
WAIT_READY → PLATE_SHOW → FEATURE_SHOW → UNSHOW_PREPARE
→ EFFECT_START → COUNT_MULT → EXPLODE → WANTED_POSTER → MATCHING_PATCH_UP (loop)
→ AWARD → ROUND_SHOW_END → ROUND_END
→ SCATTER_SHOW → FREE_PICK_ROULETTE → ENTER_FREE → ADD_FREE → FULL_REWARD → LEAVE_FREE
```

**2. Ghost Slot 機制（WantedPoster 核心改動）**

解決懸賞令退場時 `cc.Layout` CONTAINER 模式瞬間重排問題：

| 狀態 | item root active | Content active | Layout 數格？ |
|------|:--:|:--:|:--:|
| Real | ✓ | ✓ | ✓ |
| **Ghost** | ✓ | ✗ | ✓（佔位不顯示） |
| Gone | ✗ | — | ✗ |

- 2→1 時 survivor 留原位不 snap（離場者變 ghost 佔格）
- 全退（postReal=0）才釋放所有 ghost
- `NormalizeGhosts()` 為唯一收斂點（RebuildAll / HandleGameIntroComplete 起手）

**3. 事件總線通訊模式（統一範式）**

所有子系統（WantedPoster / MultiplierBase / FakeReel / FreeGameUI）透過 `eventManager.Register/Dispatch` 事件驅動，state 不直接 import 管理器。Promise wrapper 風格：payload 末尾傳 resolve callback 等動畫完成。

**4. 伺服器權威 PlateQueue**

前端不算中獎。整個消除序列由 server 一次回傳 `PlateQueue[]`（每盤含 `PlateSymbol / AwardDataVec / EliminatePos / TargetDataVec / BountyDataVec / BasePlateIndex`），前端用 `CurPlateIndex` 游標逐盤播放。

**5. Timeline 系統（新增）**

`Script/Timeline/` — 演出時間軸框架：TimelineData（定義軌道）→ TimelineExecutor（執行引擎）→ TimelinePlayer（掛在 Component 的播放器）。用於結構化管理複雜演出序列。

**6. ExtraBet two-step 變體**

`ExtraBetWindow.ts` 實作兩步驟加注選擇（先彈窗 → 確認/取消）。已知 framework 層 glitch（取消後 `ExtraBet_Active` 動畫閃一下），已記錄但選擇保持公版乾淨。
