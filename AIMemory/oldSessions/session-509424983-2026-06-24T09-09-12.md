## Daily Intel

每日情報彙整功能。Bridge 會從你配置的來源（`config/intel-sources.json` + `config/intel-profiles.json`）自動抓取資訊（YouTube 頻道、RSS、網頁等），用 agent 分析整理後產出一份「每日情報摘要」發到 Telegram。

從你的專案結構看，相關檔案：
- `src/daily-intel/` — 整個子模組（agent-analysis、browser-config 等）
- `config/intel-sources.json` — 訂閱的來源清單
- `config/intel-profiles.json` — 不同情報 profile（主題偏好、篩選規則）
- `output/daily-intel/` — 產出的報告

用途：讓你不用自己去刷各種管道，agent 幫你做資訊篩選和摘要。

---

## Embedding Router

Bridge 啟動時一次性載入本地 ONNX 模型（`bge-small-zh-v1.5`，~90MB），把所有需要語意比對的目標（topic 描述、skill 描述、wiki 摘要、指令描述、表情描述等）轉成向量。

之後每次你發訊息，只做 **1 次 embed + brute-force cosine**（<2ms），就能同時完成 7 件事：

| # | 應用 | 效果 |
|---|------|------|
| 1 | Memory recall | 根據你的訊息語意，自動注入相關 topic 的記憶 |
| 2 | Skill routing | 提示 agent 優先參考哪個 skill |
| 3 | Wiki retrieval | 自動讀取並注入相關 wiki 頁面 |
| 4 | NotebookLM routing | 語意路由到相關 notebook |
| 5 | Intent classification | 短訊息（≤25 字）提示你可能想用哪個 /指令 |
| 6 | Sticker auto-select | 短回覆（≤150 字）自動附加表情 |
| 7 | 重複 fact 偵測 | remember 時 Jaccard ≥0.7 警告重複 |

簡單講：**Embedding Router 是整個 bridge 的「語意大腦」**，讓系統不靠 keyword match、而是靠語意理解來決定該給 agent 看什麼上下文。全部跑在本地，零 API 費用，延遲 <2ms。
