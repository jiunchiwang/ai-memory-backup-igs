## P3 設計方向（因為不能撤回）

「不能抽單」代表一旦送出就是定案，因此安全等級要拉到最高：

**核心原則：雙重確認 + 不可能意外觸發送出**

1. **dry-run 產出 plan 檔**（已有構想）：含所有欄位值 + SHA256 hash + 一次性 token
2. **截圖 + ASK 確認**：agent 傳截圖給你 → 你點「確認送出」按鈕
3. **submit 階段二次比對**：重新打開表單 → 填入 → 比對所有欄位值與 plan 一致 → 才點送出
4. **送後驗證**：到 PersonalBox 確認新表單出現在列表 + 狀態正確
5. **失敗 = 停止**：任何步驟異常（欄位值不一致、頁面超時、按鈕找不到）→ 不送，回報你自行處理

**額外防線**（因為不能撤回）：
- submit 程式碼獨立在 `--submit` flag 後面，dry-run 路徑物理上碰不到
- token 一次性用完即失效，重跑 dry-run 才能再拿新 token
- 考慮加 `--confirm-text "我確認送出"` 這種 CLI 層級的二次確認（防 agent 自己偷跑）
