## Rich Messages 完整解析（Bot API 10.1，2026-06-11）

### 核心概念

Rich Messages 是 Telegram 為 AI bot 量身打造的結構化訊息格式，與舊的 `parse_mode` (Markdown/HTML) 完全不同——它是 **block-based** 架構，類似 Notion / Telegraph 的 block editor。

### 三個關鍵 API 方法

| 方法 | 用途 | 重要特性 |
|------|------|---------|
| `sendRichMessageDraft` | **串流預覽**（AI 生成中） | 暫存 30 秒的 ephemeral draft，同一 `draft_id` 動畫更新 |
| `sendRichMessage` | **最終持久化** | Draft 完成後必須呼叫此方法，訊息才會留在 chat |
| `editMessageText` + `rich_message` 參數 | **編輯** | 可後續修改已發送的 Rich Message |

### 串流工作流程（你的 bridge 最想要的）

```
1. 開始生成 → sendRichMessageDraft(chat_id, draft_id=1, rich_message={markdown: "正在思考..."})
2. 中間 chunk  → sendRichMessageDraft(chat_id, draft_id=1, rich_message={markdown: "# 結果\n部分內容..."})
3. 生成完畢 → sendRichMessage(chat_id, rich_message={markdown: "# 完整結果\n..."})
```

**關鍵點**：
- `draft_id` 相同的 draft 會**動畫過渡**（不閃爍），這就是原生 streaming！
- Draft 只存 **30 秒**，超時消失——所以長時間生成需持續呼叫
- Draft 只限 **private chat**（不支援群組）
- `<tg-thinking>` 標籤只能用在 draft 裡（顯示「思考中」動畫）

### InputRichMessage — 兩種輸入格式

```typescript
interface InputRichMessage {
  html?: string;        // HTML 格式（二選一）
  markdown?: string;    // Rich Markdown 格式（二選一）
  is_rtl?: boolean;
  skip_entity_detection?: boolean;
}
```

**Rich Markdown**（推薦，相容 GitHub Flavored Markdown）：

```markdown
# 標題

**粗體** *斜體* ~~刪除線~~ `程式碼`

```（語言高亮）

> 引用

| 表頭1 | 表頭2 |
|-------|-------|

![圖片](https://example.com/photo.jpg "圖說")

$$E = mc^2$$（LaTeX 公式區塊）

```

**Rich HTML 額外標籤**（Markdown 做不到的）：

```html

```

## 結果：grammY 完整支援 Bot API 10.1 Rich Messages

### 原生 API 支援（grammY 1.44 core）

grammY 1.44 的 `Api` class 已經包含所有 Rich Message 方法，type-safe：

```typescript
// 送 Rich Message（最終持久化）
ctx.api.sendRichMessage(chatId, { markdown: "..." }, { reply_markup: ... });

// 串流 Draft（30 秒暫存，動畫更新）
ctx.api.sendRichMessageDraft(chatId, draftId, { markdown: "..." });

// 編輯既有訊息為 Rich（text 或 InputRichMessage 二選一）
ctx.api.editMessageText(chatId, msgId, { markdown: "..." });
// ↑ 注意：editMessageText 的第三參數現在接受 string | InputRichMessage
```

### 官方 Stream Plugin（`@grammyjs/stream` v1.1.0）⭐

grammY 有**官方 streaming plugin**，直接封裝了 draft lifecycle：

```typescript
import { stream, type StreamFlavor } from "@grammyjs/stream";

bot.use(stream());

// 只要給它一個 async iterator，剩下全自動：
await ctx.replyWithMarkdownStream(asyncIterator);
// 內部自動：
//   1. 產生 draft_id
//   2. 每次 yield 呼叫 sendRichMessageDraft（動畫更新）
//   3. 30 秒 timeout 前自動 heartbeat
//   4. iterator 結束時呼叫 sendRichMessage 持久化
```

提供三種模式：
- `ctx.replyWithStream(iter)` — 純文字 streaming（用 sendMessageDraft）
- `ctx.replyWithMarkdownStream(iter)` — **Markdown Rich Message**（推薦）
- `ctx.replyWithHtmlStream(iter)` — HTML Rich Message
