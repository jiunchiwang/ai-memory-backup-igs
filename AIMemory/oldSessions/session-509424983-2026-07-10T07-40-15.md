## fable-advisor 研究摘要

### 核心概念

這是一個 Claude Code skill，用 **Fable 5 subagent** 模擬 Claude API 的 "advisor tool"（該功能是 API-only，Claude Code 裡不能用）。設計理念：

> **Opus 編排，Sonnet 做工，Fable 當顧問。**

在關鍵決策點 spawn 一個 Fable 等級的 subagent 當第二意見，而非每個 turn 都諮詢。

### 架構（兩件式設計）

| 部件 | 位置 | 功能 |
|------|------|------|
| `SKILL.md` | `~/.claude/skills/fable-advisor/` | 諮詢流程（on-demand `/fable-advisor`） |
| CLAUDE.md rule | `~/.claude/CLAUDE.md` | 自動觸發（decision point 時主動 fire） |

Skill 無法自觸發 always-on 行為，所以自動觸發必須靠 CLAUDE.md 裡的 standing instruction。

### 諮詢時機（節制是核心）

- 做不可逆決定前（寫檔 / commit / 選方案前）
- 卡住時（反覆失敗、不收斂）
- 考慮換方案時
- 大任務完成前的最終確認
- 設計 / 架構 / 風險判斷

**不諮詢**：短回覆、機械性編輯、簡單查找。一般一個多步驟任務只諮詢 2 次。

### 兩種諮詢模式

1. **Focused brief**（預設）：自己打包 context 給 Fable，請它在 ~150 words 內給方向。適合設計/架構/判斷。
2. **Read-tools investigation**：給 Fable read-only agent（Explore / general-purpose），讓它自己讀 code 再建議。花更多 token，但適合 code review 或細節密集的決策。

### 建議權重

- Fable 建議給**高權重**（更強模型 + 新鮮視角）
- 只在**經驗性失敗**或**一手資料矛盾**時才不採納
- 自己的 data 跟 Fable 矛盾 → 再開一次 focused Fable call 對質，而非靜默切換

### 限制

- **只在 Opus-main session 使用**（Fable→Fable 是浪費）
- 需要 Fable tier 存取權（Anthropic API only，Bedrock/GCP/Foundry 不行）
