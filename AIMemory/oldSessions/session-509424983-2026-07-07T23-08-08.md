### 🤖 Assistant (上午7:01:42)

核心差異：**working state 是 agent 主動寫的「待辦交接」，archive 是 bridge 自動蒸餾的「對話快照」**。

| | Working state | Session archive |
|---|---|---|
| 誰產生 | Agent 自己用 `remember()` 寫 `[WS]` 條目（restart 前或里程碑時） | Bridge 在 session 關閉時自動生成（`buildArchive`），agent 不用做任何事 |
| 內容 | 結構化任務欄位：task / completed / blocked / key_refs / next_action | Session 元資料（時長、輪數、關閉原因、goal 狀態、字數統計）+ LLM 全程摘要（沒有就 fallback 最近幾則 assistant 回覆節錄） |
| 語意 | 「接下來該做什麼」— 前瞻性，指向 `next_action` | 「剛剛發生了什麼」— 回顧性，補對話脈絡 |
| 儲存 | `working-state-{chatId}.md`（`sessionManager.ts:317`） | `archive JSON` per-chatId 單檔覆寫（`session-archive.ts:177`） |
| 可靠性 | agent 忘了寫就沒有（所以 preamble 提醒增量存檔） | 一定有——crash / idle / manual close 都會產生 |
| 消費後 | 注入一次即 `unlink`（`:326`） | 注入一次即 `deleteArchive`（`:339`），都是 one-shot |

兩者互補：archive 保底（自動、但只有摘要層級）；working state 精準（有明確 next_action、但靠 agent 自律）。像這次上個 session 是手動關閉、agent 沒寫 WS，所以 working state 只有「session closed」空殼，實際續接靠的是 archive 的 `Recent assistant context`。
