# Localhost Tunnel 方案研究報告（2024–2026）

目標：把 Windows 本機 `http://localhost:3847` 暴露為公網 HTTPS URL，供 Telegram Mini App 使用，需支援 SSE 長連線。

## 1. ngrok

| 項目 | 內容 |
|------|------|
| **SSE 支援** | ✅ 完整支援。ngrok 是 L7 HTTP proxy，不對長連線設 idle timeout（free tier 明確標明 "no timeout on endpoints"）。SSE 正常運作。 |
| **免費額度** | 1 static dev domain（自動分配，如 `xxx.ngrok-free.app`）、20,000 HTTP requests/月、1 GB 頻寬/月、3 concurrent endpoints、**無 session 時間限制** |
| **Windows 安裝** | `winget install ngrok`、`choco install ngrok`、或下載 zip 解壓。Node.js SDK：`npm install @ngrok/ngrok` |
| **程式化啟動** | **官方 Node.js SDK `@ngrok/ngrok`**（無需外部 binary）— 3 行 code 即可取得 URL |
| **啟動速度** | ~2–5 秒取得 URL |
| **穩定性** | ⭐⭐⭐⭐⭐ 商業級服務，全球 CDN edge，極少斷線 |
| **已知問題** | ① Free tier 有 interstitial page（瀏覽器訪問時彈「Visit Site」頁面），但 API/SSE 請求不受影響（非 HTML 不觸發）。② Telegram Mini App 的 WebView 首次載入可能看到 interstitial — 需加 `ngrok-skip-browser-warning` header 或用 custom User-Agent。③ 20k requests/月可能在高頻使用下不夠。 |

**程式化啟動範例：**
```typescript
import ngrok from '@ngrok/ngrok';

const listener = await ngrok.forward({
  addr: 3847,
  authtoken_from_env: true, // reads NGROK_AUTHTOKEN
});
const url = listener.url(); // https://xxx.ngrok-free.app
```

## 2. Tailscale Funnel

| 項目 | 內容 |
|------|------|
| **SSE 支援** | ✅ 支援。Funnel 做 HTTPS TLS termination + reverse proxy 到本機，不截斷長連線。 |
| **免費額度** | 完全免費（Tailscale Personal 方案 100 台設備以下免費），Funnel 無流量/請求限制 |

| **程式化啟動** | 需用 CLI `tailscale funnel 3847`（spawn child process）。無官方 Node.js SDK。 |
| **啟動速度** | ~3–8 秒（首次需 provision TLS cert，後續秒級）|
| **穩定性** | ⭐⭐⭐⭐ 商業級服務，WireGuard 底層，穩定 |
| **已知問題** | ① **域名固定**為 `machine-name.tailnet-name.ts.net`，不可自訂隨機。② 需要 Tailscale daemon 一直運行。③ 需先 `tailscale funnel --https=443 localhost:3847` 啟用。④ **公司網路可能封 WireGuard/DERP 連線**（和 cloudflared QUIC 類似問題）。⑤ 不能純 programmatic — 需系統級 Tailscale 安裝。 |

```typescript

```

### 🥇 第一名：ngrok（`@ngrok/ngrok` Node.js SDK）

**理由：**
- **唯一提供官方 Node.js SDK，無需外部 binary**——完美契合「程式化啟動」需求

**實現方式（替代現有 cloudflared）：**
```typescript
import ngrok from '@ngrok/ngrok';

export async function startTunnel(port: number): Promise<string | undefined> {
  if (process.env.STATUS_HTTPS_URL?.trim()) {
    return process.env.STATUS_HTTPS_URL.trim();
  }
  try {
    const listener = await ngrok.forward({ addr: port, authtoken_from_env: true });
    const url = listener.url()!;
    process.env.STATUS_HTTPS_URL = url;
    return url;
  } catch (e) {
    console.warn('[tunnel] ngrok failed:', (e as Error).message);
    return undefined;
  }
}
```
