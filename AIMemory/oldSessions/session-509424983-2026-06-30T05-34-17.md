## Phase B+：Specialist Persistent Memory（~50 行）

### 目的

目前 specialist 分身（slot-dev、researcher、general）每次被 spawn 時都是「白紙一張」——沒有上一次任務的記憶。Phase B+ 讓分身**跨 session 累積知識**，避免：

- 每次重複交代同樣的偏好/約束
- 同樣的錯誤反覆犯（沒有 lessons learned）
- delegation 產出品質無法隨時間提升

概念靈感來自 Claude Code 的 Multi-Agent 記憶設計：subagent 結束時自動把 key takeaways 寫回上層。

### 作法

利用 **Phase B 已完成的 PostToolHook registry** 作為基礎，註冊一個 hook：

```typescript
// 概念（~50 行）
registerHook(async (event) => {
  // 只攔截 specialist session 結束事件
  if (event.name !== "specialist_session_end") return;
  
  // 從 specialist 最終回覆中抽取 key facts
  const { specialistName, summary, lessonsLearned } = extractFromOutput(event.output);
  
  // 追加到 specialist 專屬記憶檔
  // e.g. ${MEMORY_DIR}/specialist-memory/<name>.md
  appendToSpecialistMemory(specialistName, { summary, lessonsLearned });
});
```

**資料流：**
1. Specialist 完成任務 → artifact 已存（現有機制）
2. Hook 攔截 → 從 artifact/output 抽取 lessons（如「這個專案用 X 不用 Y」）
3. 寫入 `${MEMORY_DIR}/specialist-memory/<name>.md`
4. 下次 spawn 同一 specialist 時，preamble 注入這份記憶

### 與現有機制的差異

| | 現在 | Phase B+ 後 |
|--|------|------------|
| Specialist 記憶 | 無（每次全新） | 跨 session 累積 |
| 存放位置 | artifacts/（結構化 JSON，但不回注） | specialist-memory/（下次 preamble 注入） |
| 觸發方式 | 手動 | PostToolHook 自動 |
