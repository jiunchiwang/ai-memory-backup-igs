## 方案 B：Pluggable PostTool Hooks（~80 行）

**概念**：在 `tool-loop.ts` 加入 hook registry，讓各模組自行註冊 post-tool handler。

### Interface

```typescript
// tool-loop.ts 新增 interface
export interface PostToolHook {
  /** Glob-style matcher: "*", "shell", "codegraph_*", "mcp__memory__*" */
  matcher: string;
  /** Called after tool execution. Can modify output or inject hints. */
  handler: (ctx: {
    name: string;
    args: Record<string, unknown>;
    output: string;
    isError: boolean;
    messages: ChatMessage[];  // 可讀可寫
  }) => void | Promise<void>;
}

// 在 runToolLoop 參數加入
export interface ToolLoopOptions {
  // ...existing...
  postToolHooks?: PostToolHook[];
}
```

### 執行點（`executeToolCalls` 完成後）

```typescript
// tool-loop.ts — 在 results append 到 messages 之後
for (const r of results) {
  messages.push({ role: "tool", tool_call_id: r.id, content: r.output });
  
  // 跑 matching hooks
  for (const hook of (opts.postToolHooks ?? [])) {
    if (matchGlob(hook.matcher, r.name)) {
      await hook.handler({ name: r.name, args: r.args, output: r.output, 
                           isError: r.isError, messages });
    }
  }
}
```
