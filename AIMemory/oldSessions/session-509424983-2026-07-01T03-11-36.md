## Claude Sonnet 5 重點整理

**發布日期**：2026-06-30（昨天）

### 定位

Sonnet 5 是目前最 agentic 的 Sonnet 模型——規劃、工具使用、自主執行能力接近 Opus 4.8，但價格更低。相比前代 Sonnet 4.6 在 reasoning、tool use、coding、knowledge work 全面提升。

### 定價

| | Input | Output |
|--|--|--|
| 優惠價（至 2026-08-31） | $2/MTok | $10/MTok |
| 正式價（9月起） | $3/MTok | $15/MTok |

對比：Opus 4.8 是 $5/$25。

⚠️ 注意：Sonnet 5 用了新 tokenizer，同樣文字會多 1.0–1.35× tokens，優惠價設計上讓轉換大致 cost-neutral。

### 效能亮點

- BrowseComp（agentic search）與 OSWorld-Verified（computer use）上嚴格優於 Sonnet 4.6，在高 effort level 下部分場景能追平 Opus 4.8
- 早期用戶反饋：能完成之前 Sonnet 會中途放棄的複雜任務、會主動 self-check output、coding + debugging 更持久

### 安全

- 整體 misaligned behavior 低於 Sonnet 4.6（更安全）
- 幻覺率和 sycophancy 率下降
- agentic safety 改善（更好地拒絕惡意請求、抵抗 prompt injection）
- 網安能力顯著弱於 Opus 4.8/Mythos 5（刻意沒在 cyber 任務上訓練）
- 預設啟用 cyber safeguards

### 可用性

所有 plan 立即可用（Free/Pro/Max/Team/Enterprise）、Claude Code、Claude Platform。API model name：`claude-sonnet-5`。

---

### 對你的 bridge 的影響

你目前主 agent 跑 `claude-opus-4.6`，specialist 跑 `claude-sonnet-4`。考量點：

1. **Specialist 升級候選**：Sonnet 5 的 agentic 能力大幅提升，specialist（slot-dev / researcher / general）從 sonnet-4 升到 sonnet-5 應該能改善 delegation 完成率，且在優惠期間成本差不多。

2. **主 agent 是否降級**：如果 Sonnet 5 在高 effort 下能追平 Opus 4.8，某些場景可以考慮用 Sonnet 5 取代 Opus 4.6 省成本——但對於你重度使用的長 session + 複雜 tool orchestration，Opus 的推理深度可能仍有優勢。

3. **Tokenizer 變更**：同樣 preamble 會多吃 0–35% tokens，context budget discipline 更重要了。
