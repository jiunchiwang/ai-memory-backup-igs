### Hermes 是什麼？

Hermes 是一款**長駐型 AI Agent**，可安裝在本機電腦或雲端 VPS，透過 Telegram 操控。跟 ChatGPT / Claude 等對話式 AI 的差異在四個核心特色：

1. **長時間待命** — 背景獨立運作，可隨時透過手機 App 交辦任務
2. **自我學習** — 完成任務後回顧改進，下次更熟練
3. **分層記憶系統** — 記住偏好、工作模式、過去交辦事項（User.md + Memory.md + Soul.md）
4. **安全機制** — 危險指令審查、刪檔前詢問授權

---

### 安裝方式（影片示範兩種）

| 方案 | 優點 | 缺點 |
|------|------|------|
| **本機安裝** | 幾乎免費、被駭風險低 | 關機/休眠就離線 |
| **雲端 VPS（Hostinger）** | 24h 穩定、不碰私人檔案 | 每月付費 |

- 雲端安裝：Hostinger 後台直接選 "Hermes Agent" 一鍵部署
- 本機安裝：建議新增獨立使用者帳號（非管理員），隔離權限

---

### AI 模型選擇

- **訂閱制**：直接用 OpenAI（ChatGPT）的訂閱
- **用量制**：透過 **OpenRouter** 儲值，自由切換各種模型（示範 DeepSeek、GPT-5.5 等）

---

### 串接 Telegram（核心操作介面）

1. 在 Telegram 找 @BotFather → `/newbot` 建立機器人
2. 取得 Bot Token 貼回 Hermes
3. 設定 User ID 白名單（防外人偷用）
4. 設定 Home Channel

### 功能展示（以虛擬角色「小芳」電商創業為例）

| 功能 | 說明 |
|------|------|
| **記憶系統** | User.md（個人檔案）、Memory.md（經驗筆記）、Soul.md（人設語氣） |
| **對話壓縮** | 偵測對話過長自動壓縮，不丟重點 |
| **網頁搜尋** | 內建搜尋 + 可串 Tavily 做深度研究 |
| **Subagent（子代理）** | 同時派出多個分身並行處理子任務 |
| **Google 整合** | Gmail + Calendar + Drive + Docs（透過 OAuth） |
| **排程** | 每日晨報、定時推送學習內容、定時備份 |
| **語音模式** | `/voice` 啟用，Telegram 錄音→語音回覆 |
| **Skill（技能/SOP）** | 自訂流程（如：拍收據→裁切→OCR→填入 Google Sheet）；內建 70+ skills；官方目錄可安裝社群 skill |
| **生圖/生影片** | 透過 FAL.ai 串接（GPT Image 2、Seedance 2.0） |
| **Notion 串接** | 透過 API 把商品照搬入 Notion + 自動生文案 |
| **Goal 模式** | 設定大目標，Hermes 自動拆解成多步驟逐輪推進 |
| **備份** | 排程每日凌晨打包到 Google Drive |
| **防火牆** | Hostinger 後台設定，保護公開主機 |

### 與你的 Telegram-Kiro-Bridge 的對比

看完這部影片，Hermes 的架構跟你做的 telegram-kiro-bridge 有很多相似之處：

| 概念 | Hermes | 你的 Bridge |
|------|--------|-------------|
| 長期記憶 | User.md / Memory.md | facts-*.md + topic shards + wiki |
| 人設 | Soul.md | preamble + agent profile |
| Skill/SOP | 內建 70+ 可裝社群 | ~/.kiro/skills/ + SKILL_PROPOSE |
| 排程 | 內建 schedule | /schedule + <<SCHEDULE>> token |
| 子代理 | Subagent | Specialist + PARALLEL_DELEGATE |
| Goal | Goal 模式 | /goal 持續性目標 |
| 備份 | 排程打包到 GDrive | /backup robocopy + git push |
| 語音 | /voice 模式 | 目前沒有（Telegram voice 可考慮） |
| 模型切換 | OpenRouter | ACP_AGENT_COMMAND 切 agent |
| 安全 | 危險指令審查 | ALLOWED_USER_IDS + TG_SEND_ROOTS 白名單 |

**結論**：Hermes 是一個面向非工程師的「開箱即用」AI Agent，強調簡單設定 + Telegram 操控。你的 bridge 在架構上更進階（ACP 協定、MCP 工具生態、embedding router、跨機 relay），但兩者解決的核心問題一樣——**讓 AI Agent 長駐 + 手機操控 + 記憶持久化**。Hermes 的 Skill 市集和語音模式是值得參考的方向。
