## Ruflo 是什麼

GitHub 46.6k stars 的開源 AI 多代理編排平台，由 ruvnet 開發，把 Claude Code 擴展成多 agent swarm。核心賣點：

1. **多代理 swarm** — 把單一 Claude 擴成多 worker 並行
2. **AgentDB 持久記憶** — HNSW 向量搜尋跨 session
3. **成本路由** — 按任務複雜度分配到不同模型（宣稱省 75%）
4. **313 個 MCP 工具**（實際只有 ~195 個能用）

## 架構亮點

- **Hive-Mind**：Queen（決策）+ Worker（執行）階層，4 種拓撲、4 種共識協議
- **SONA 自學**：每次任務完成後回饋 routing 決策
- **SPARC → Skills 轉向**：從固定 5 階段流程改為可組合能力自決序

## 真相（自審報告揭露）

| 問題 | 影響 |
|------|------|
| Agent execution 未接線 | spawn 只寫 Map，不呼叫 LLM |
| Hive-mind 單進程 | 共識協議是裝飾品 |
| Workflow runtime 缺失 | `workflow_execute` 回 "not found" |
| WASM agent 只是 echo | 零加速 |

**結論：~240 工具中只有 ~195 真正運作。**

## 與我們的 bridge 比較

| 面向 | Ruflo | telegram-kiro-bridge |
|------|-------|---------------------|
| 多 agent | swarm（宣稱分散式，實為單進程） | specialist + relay（本機 stdio + 跨機 Telegram） |
| 記憶 | AgentDB（HNSW） | facts + topic shard + embedding router（bge-small-zh） |
| 成本路由 | model 複雜度路由 | `/model` 手動 + trio-llm 本機優先 |
| 工具整合 | 313 MCP 工具（195 真實） | 內建 memory MCP + 外掛任意 MCP server |
| 穩定性 | alpha，多處 stub | 生產穩定，每日 /dream 維運 |
