### 👤 User (上午6:15:48)

命令 handoff 和 reset 差在哪？

---

### 🤖 Assistant (上午6:16:01)

核心差異：

| | `/reset` | `/handoff` |
|--|----------|-----------|
| **目的** | 單純砍掉 session 重開 | 「收工交接」— 先整理再結束 |
| **做的事** | 直接關閉 ACP session，下則訊息開新 session | 產生 handoff 摘要、把重要內容寫入長期記憶候選，然後才結束 session |
| **記憶處理** | 不做任何記憶萃取 | 自動讓 agent 抽事實/決策/上下文，確保下次接續時有跡可循 |
| **選項** | 無 | `keep`（不關 session）、`backup`（接著跑備份） |

簡單比喻：
- `/reset` = 直接關機重開
- `/handoff` = 下班前寫交接單、存檔，然後才關機

用 `/handoff` 的時機：這個 session 做了有價值的事，你想確保下次（或隔天 dream 後）能無痛接續。用 `/reset` 的時機：session 卡住、context 髒了、單純想要乾淨環境。

- App 裡找「新增/匯入保單」功能，用保單號碼 + 身分證字號綁定
