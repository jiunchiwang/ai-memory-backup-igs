**Webwright** 是微軟研究院（Microsoft Research + HKU）2026 年 5 月發布的開源框架，核心理念是：

> 給 LLM 一個 terminal，讓它自己寫 Playwright 程式碼來操作瀏覽器，而不是傳統的「看截圖 → 預測下一個點擊」。

### 架構特點

- **極簡 harness**：只有 ~1K 行程式碼，三個模組（Runner、Model Endpoint、Environment），無複雜多 agent 編排
- **一次性瀏覽器**：agent 可以隨時 spawn / 關閉瀏覽器 session，不被困在單一頁面狀態
- **產出是可重用程式**：完成任務後留下的是 Python script，可以參數化後變成 CLI 工具反覆使用
- **自我驗證**：agent 必須在乾淨資料夾重跑 final script + 通過 self-reflection 才算完成

### 效能

| Benchmark | 分數 |
|-----------|------|
| Odysseys（長程瀏覽） | 60.8%（比前 SOTA 高 35%） |
| Online-Mind2Web（300 個真實網站任務） | 86.7% |

### 整合方式

它提供 `.claude-plugin` 和 `.codex-plugin`，也有 `skills/webwright/` 資料夾——意味著可以作為 **Codex / Claude Code 的 skill** 掛載。

#### 方案 A：作為 Kiro skill（最自然）

#### 方案 B：作為 Specialist 的能力（推薦）

```jsonc

}
```

Codex 原生支援 `/plugin install webwright@webwright`，researcher specialist 用 Codex harness 就能直接呼叫 Webwright。

**優點**：
- Codex plugin 系統原生支援，零 bridge 改動
- Specialist 隔離，不影響主 agent
- 透過 `<<RELAY_DELEGATE:researcher|去查 XXX>>` 委派

**缺點**：
- 需要額外裝 Codex CLI + `codex-acp`
- 需要 OpenAI API key（或設定 Codex 用 Anthropic）

#### 方案 C：作為獨立 CLI 工具手動呼叫

不整合進 agent 系統，純粹當一個本機工具：


**優點**：最簡單、零整合成本
**缺點**：不自動、不在 agent 工作流裡

## 影片主題：微軟 SkillOpt — 讓凍結模型性能提升 23.5%

**論文**：arXiv:2605.23904（2026 年 5 月）
**核心概念**：把 SKILL.md 當作「可訓練的外部參數」，用類似梯度下降的方式迭代優化，模型本身完全不動。

## 一句話總結

> 不改模型權重，只改 skill 文件，用「提出小修改 → 跑測試 → 有進步才接受」的循環，把一份 markdown 指引「訓練」到最佳狀態。

---

## 運作流程

```
初始 SKILL.md
    ↓
用凍結模型跑一批任務（rollout）
    ↓
另一個 optimizer 模型分析失敗軌跡
    ↓
提出有限度的文字修改（≤4 edits/step）
    ↓
在 held-out 驗證集測試 → 有進步？
    ├─ Yes → 接受，更新 best_skill.md
    └─ No  → 丟進 rejected buffer，不重複犯錯
    ↓
重複直到收斂 → 部署最終 skill（零額外推理成本）
```

### 五件套觀念

---

### 心態轉換（最重要）

1. **你是品檢人員，不是程式設計師** — 驗收，不是手把手教
2. **描述目標，不描述步驟** — 讓 AI 自己想怎麼做
3. **一次給完整需求** — 不要分 10 次對話
4. **用「期望」不用「指令」** — AI 會在期望中越來越接近你的標準
5. **痛苦過程有價值** — 你的 KnowHow 就是你的不可替代性

---

### 6 大提詞模板

1. **4 連問** — 進入新主題快速建立全貌
2. **規格反推** — 把舊文件變 AI 可讀
3. **一次到底** — 不要中途停下來問「繼續嗎」
4. **自評提詞** — 強制打分，< 85 自己改
5. **繼續優化** — 「繼續檢視可優化點」跑 6-7 輪收斂
6. **期望式 5 句型** — 我希望 / 強制 / 你的身分是 / 請參考 / 過程中遵守

---

### 雙 Skill 自省循環（威力最強）

```
Generator 產出 → QA 找 5+ 問題 → Generator 修正 → 重複到 ≥95 分
```

核心：**做的人和檢查的人用不同 prompt**，避免自我偏袒。
