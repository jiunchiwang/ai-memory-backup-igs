## DreamCoder — Claude Code 的開源桌面版 GUI

**定位**：把 Claude Code（純 CLI 工具）包成現代原生桌面 App，讓你用圖形介面管理 session、切換模型、處理檔案。

**技術棧**：
- 桌面外殼：Tauri 2（Rust）
- 前端：React 18 + Vite + TailwindCSS 4
- 後端 runtime：Bun（Node.js 相容）
- 終端：portable-pty (Rust) + xterm.js
- 狀態管理：Zustand
- 協議：WebSocket / MCP / LSP

**核心功能**：
1. **會話管理** — 視覺化歷史、側邊欄、多 tab
2. **整合終端** — 內建 PTY，集成 xterm.js
3. **多模型切換** — 支援 Anthropic / OpenAI / DeepSeek / 通義 / MiniMax / Azure / Google Vertex / AWS Bedrock
4. **Computer Use** — 原生截圖模式 + UIA Tree 文本輔助模式
5. **MCP 支援** — 圖形化配置 MCP server
6. **隱私優先** — API Key 和資料完全本地，不依賴雲端

**平台狀態**：
- ✅ Windows x64（主要維護平台，有預編譯安裝包）
- ⚠️ macOS / Linux 有構建腳本但維護者未實測

**Roadmap 重點**：Phase 3 計劃做 H5 遠端存取（手機瀏覽器接入桌面 session），Phase 4 計劃做 IM 適配器（飛書/釘釘/Telegram/微信）。

**最新版本**：v0.4.0（2026-06-14，"Phone-in-your-pocket release"）

**授權**：MIT

## DrawCall 和 Triangle 的計算方式

這個插件的統計數據在 `src/panels/default/index.ts` 第 ~742-756 行，每 15 幀更新一次：

### Triangle（三角形數）

```typescript
let tris = 0, meshCount = 0, visibleSlots = 0;
for (const slot of skeleton.drawOrder) {
    const att = slot.getAttachment();
    if (!att) continue;
    visibleSlots++;
    if (att.triangles) { tris += att.triangles.length / 3; meshCount++; }
    else if (att.region) tris += 2;  // 一個 region attachment = 一個矩形 = 2 個三角形
}
```

**邏輯**：
1. 遍歷 `skeleton.drawOrder`（當前幀可見的 slot 排序）
2. 每個 slot 取 attachment：
   - **MeshAttachment**（有 `triangles` 陣列）→ `triangles.length / 3`（因為每 3 個 index = 1 個三角形）
   - **RegionAttachment**（有 `region`，即普通矩形圖片）→ 固定 **2** 個三角形（一個 quad = 2 tri）
   - 無 attachment → 跳過

### DrawCall

```typescript
const dc = (batcher as any).drawCalls;
```

這個值來自 `spine.webgl.PolygonBatcher`（spine-webgl.js 第 9124-9169 行）：

```javascript
// begin() 時歸零
PolygonBatcher.prototype.begin = function (shader) {
    this.drawCalls = 0;
    // ...
};

// 每次 flush() 時 +1
PolygonBatcher.prototype.flush = function () {
    // ... 實際 gl.drawElements 呼叫
    this.mesh.draw(this.shader, gl.TRIANGLES);
    this.drawCalls++;
};
```

**觸發 flush（= +1 DC）的條件**：
1. **Texture 切換** — `draw()` 時發現新 texture ≠ 上一個 texture → flush 舊 batch
2. **Buffer 滿** — vertex 或 index 超過上限（10920 三角形）→ 強制 flush
3. **BlendMode 切換** — `setBlendMode()` 呼叫時 flush
4. **End** — `end()` 時把剩餘的 flush 掉

### 渲染流程

```
batcher.begin(shader)
  → renderer.draw(batcher, skeleton)   // spine SkeletonRenderer 逐 slot 繪製
      → 每個 slot 呼叫 batcher.draw(texture, vertices, indices)
      → texture 變了或 buffer 滿就自動 flush（DC+1）
batcher.end()                          // 最後一批 flush（DC+1）
```

### 簡單結論

| 指標 | 意義 | 怎麼降低 |
|------|------|---------|
| **Triangle** | 當前幀所有可見 slot 的三角形總和 | 減少 mesh vertex、用 region 代替不需要變形的部件 |
| **DrawCall** | 一幀內 GPU 被提交幾次繪製命令 | 合併 atlas（同一張 texture 不斷 batch）、避免頻繁切換 blend mode |

典型情況：如果所有 slot 都用同一張 atlas + 同一個 blend mode，整個 skeleton 只會產生 **1 個 DrawCall**。每多一張不同的 texture 或切一次 blend mode 就多一個 DC。

### 🤖 Assistant (下午8:51:50)

是的，它是**執行時（runtime）即時計算**的，不是靜態分析。

具體來說：

1. **Triangle** — 每 15 幀從 `skeleton.drawOrder` 讀取**當前幀**的 slot 可見狀態。因為 Spine 動畫會隨幀切換哪些 slot 可見、哪些 attachment 被掛上，所以三角形數會**隨動畫播放而變動**（例如某個 slot 在特定幀隱藏，tris 就少了）。

2. **DrawCall** — 是 `batcher.begin()` → `renderer.draw()` → `batcher.end()` 這個**實際渲染流程**跑完後的結果。`drawCalls` 計數器在 `begin()` 歸零、每次 `flush()` 加一，所以讀到的是**那一幀真正提交給 WebGL 的 draw call 數**。

兩個值都不是預先算好的，而是插件的 canvas **每幀實際跑 spine-webgl 渲染管線後**才取得的 live 數據。你在面板上看到的數字會隨著：
- 當前播放的動畫（不同 animation 可見 slot 不同）
- 當前 skin（不同 skin attachment 不同）
- 動畫進行到哪一幀（有些幀可能隱藏/顯示 slot）

而即時變化。
