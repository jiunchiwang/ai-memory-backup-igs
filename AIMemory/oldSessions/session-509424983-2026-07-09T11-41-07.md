**結論：不需要本地 slot 專案也能正常運作。**

| 面向 | 狀態 |
|------|------|
| 硬編碼絕對路徑 | **零** — 沒有 `E:\`、`F:\` 等路徑 |
| 專案名稱引用 | 有（eye_strike、722、746 等）— 但都是**資訊性**（「此模式來自 eye_strike」），不是操作性（「讀取 E:\UK\eye_strike\...」） |
| 執行依賴 | **無** — 沒有腳本、沒有 import、不需要任何專案在磁碟上存在 |

每張 pattern 卡片是自包含的蒸餾知識（識別條件 / State 映射 / 演出時序 / 邊界案例），agent 讀完卡片就能做模式比對和實作骨架產出，不需要去翻參考專案的原始碼。

### 🤖 Assistant (下午7:34:35)

基於實測（uk_917 probe）和剛修完的 5 個 bug，我看到幾個層次的優化空間：

## 1. 架構層：spec_adapter 的定位尷尬

codegen 自帶 `spec_adapter.py`，但 spec-to-impl 用的是 `excel-to-ai-doc`（抽圖、逐格保真）。兩條路平行存在：

```
xlsx ──→ spec_adapter.py ──→ Game_Spec.md ──→ codegen LLM
xlsx ──→ excel-to-ai-doc  ──→ markdown+圖  ──→ spec-to-impl
```

spec_adapter 的三種 layout parser（A/B/C）靠 label 猜格式，**每碰到新排版就要加 parser**，且不抽圖——規格書一半資訊量在圖裡。

**優化方向**：codegen 改吃 excel-to-ai-doc 的輸出，廢掉 spec_adapter。省去維護多套 xlsx parser 的成本。

## 2. 可攜性：硬編碼路徑

| 位置 | 硬編碼 |
|------|--------|
| `SKILL.md:119` | `E:\UK\uk_slot_template\` |
| `check_regression_v2.py:275` | `E:/UK/uk_slot_eye_strike_new` |
| `post-codegen-extract.ts:11` | `E:\UK\uk_slot_template` |
| `ensure_game_meta.py:20` | `E:\UK\uk_slot_framework\templates\` |

換台機器就壞。應改成參數或環境變數（像 bridge 的 `MEMORY_DIR` 做法）。

## 3. 知識蒸餾不足

pattern-library 完全自包含，codegen 的 `_api-ref.md` 只蒸餾了部分 template API。實際執行時 agent 仍要讀 template 原始碼才能產 code——等於**把 template 的存在當隱性依賴**。

如果把 template 的骨架結構（state machine 接線、SlotReels 配置、AudioManager 用法）也蒸餾成 reference doc，codegen 就能脫離本地 template 運作。

## 4. update mode 形同虛設

anchor merge 只認 `<<CODEGEN_BEGIN:name>>` 標記的區塊，手寫專案沒有 anchor → 全部被當「無標記代碼」覆寫。**實際上只有 new mode 有用。**

兩個選擇：
- 砍掉 update mode（誠實標為 new-only tool）
- 改成 diff-based patch 而非 anchor-based replace

## 5. 執行覆蓋盲區（結構性問題）

agent 按需讀 `_flow.md` 各 Step 段落，**但不讀 SKILL.md 的慣例覆蓋**。我們加了 Proto.ts 注記到 Step 3.3 修了一處，但同樣的問題可能在其他 Step 重複出現。

**根本解法**：慣例不放 SKILL.md，全部就地寫進 `_flow.md` 對應 Step。或在每個 Step 開頭加 `> 前置：讀 _pitfalls.md §X.X` 指引。

## 6. 優先級建議

| 優先級 | 項目 | 理由 |
|--------|------|------|
| **P0** | 廢 spec_adapter，改吃 excel-to-ai-doc | 減少維護面 + 品質提升（抽圖） |
| **P0** | 路徑參數化 | 換機器就壞，阻礙協作 |
| **P1** | 蒸餾 template API 到 reference doc | 減少對本地專案的隱性依賴 |
| **P1** | 慣例覆蓋下沉到 _flow.md 各 Step | 堵覆蓋盲區 |
| **P2** | 砍或重做 update mode | 目前形同虛設 |
