### 🤖 Assistant (下午10:12:52)

在 AI agent 的語境下，**governance** 指的是「治理 / 管控機制」——確保 agent 不會失控做出不該做的事。

具體來說，以 Claude Opus 4.8 的 dynamic workflow 為例：

- **誰能下令**：哪些任務可以自動分配給 subagent，哪些需要人類審批
- **權限邊界**：subagent 能讀寫哪些檔案、能呼叫哪些 API、能不能做不可逆操作
- **監督與可見性**：執行過程有沒有 log、人類能不能中途介入或終止
- **安全護欄**：防止 agent 連鎖錯誤（例如你 bridge 的 loop guard、turn budget、cooldown）

對應到你的 bridge 系統，你已經有類似的 governance：
- Goal turn budget（1-50）防無限跑
- Specialist parallel delegate 完成後才一次注入結果
- `<<RESTART>>` 有 cooldown 5min + cap 5 次
- Relay loop guard 連續 10 turn 就封鎖

簡單說：governance = 「讓 agent 有能力做事，但在框架內做事」的那套規則和機制。
