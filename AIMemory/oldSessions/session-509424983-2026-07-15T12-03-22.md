## MoA（Mixture of Agents）功能介紹

MoA 是 bridge 內建的**多模型盲審/辯論/規劃引擎**——讓多個 LLM（reference）獨立回答同一個問題，再由主 agent 綜合出最終結論。

---

### 核心概念

```
你的問題
  ├── Reference A（Claude）──┐
  ├── Reference B（Kiro）────┼── 各自獨立回答（互看不到）
  └── Reference C（Codex）───┘
                          ↓ 收集所有意見
                    主 Agent 綜合成最終回覆
```

- Reference 之間**互不可見**（盲審），確保獨立性
- Reference 是「閹割版 specialist」——不給 MCP、不能寫檔，只回答問題（Phase 4 後可讀檔）
- 主 agent 作為 Aggregator，擁有完整 context + 工具，負責綜合判斷

---

### 指令用法

| 語法 | 說明 |
|------|------|
| `/moa <問題>` | 自動路由 preset（embedding 語意匹配） |
| `/moa <preset> <問題>` | 明確指定 preset |
| `/moa:red-team <問題>` | 冒號語法指定 preset |
| `/moa plan <任務>` | 進入規劃模式（多模型協作拆任務） |
| `/moa` | 顯示可用 presets + 用法 |

---

### 可用 Presets（你的實際配置）

| Preset | References | 用途 |
|--------|-----------|------|
| `default` | Claude + Kiro | 泛用盲審：方案評估、設計 review、架構討論 |
| `red-team` | Claude + Adversary | 安全/風險角度找漏洞、devil's advocate |
| `fact-check` | Claude + Kiro | 事實查核：比對一致/矛盾/單方提到 |

路由是 embedding 自動判斷——問安全相關的問題會自動走 `red-team`，問「這樣對嗎」會走 `fact-check`。

---

### 使用範例

**1. 盲審設計方案**
```
/moa 我想把 session archive 改成 append-only 而不是覆寫，優缺點？
```
→ Claude 和 Kiro 各自獨立分析 → 主 agent 綜合

**2. 紅隊安全審查**
```
/moa:red-team 新的 /sendfile 白名單機制有漏洞嗎？
```
→ Claude 正常分析 + Adversary 專門找攻擊面 → 主 agent 彙整風險

**3. 事實查核**
```
/moa:fact-check Telegram Bot API 的 sendDocument 最大檔案限制是 50MB 對嗎？
```
→ 多模型各自回答 → Aggregator 比對一致性，標記矛盾

**4. 規劃模式（Phase 2）**
```
/moa plan 建一個完整的使用者登入系統
```
→ 多模型各自提出拆法 → Aggregator 綜合成結構化計畫 → 確認按鈕（開始/修改/取消）→ 自動 PARALLEL_DELEGATE 執行

---

### 顯示效果（Telegram）

```
🧠 MoA 盲審中... (2 位顧問)

━━━ Claude 的看法 ━━━
我認為 append-only 的好處是...
但風險在於...

━━━ Kiro 的看法 ━━━
整體可行，另外建議注意...

━━━ 最終綜合 ━━━
[主 agent 基於兩方意見做出的判斷]
```

---

### 自動觸發（/goal 進行中）

在 `/goal` active 期間，以下情況會自動觸發 MoA red-team：

1. 一個 turn 改了 ≥5 個檔案 → 事後 review 該批改動
2. Agent 連續失敗 ≥2 次（reflexion 卡住）→ 請第二意見

護欄：同 goal 最多 3 次、間隔 ≥10 分鐘，防額度燒光。

---

### Plan Mode 細節

`/moa plan <任務>` 進入規劃模式後：

1. Reference 各自提出任務拆解
2. Aggregator 綜合成結構化 JSON（`<<MOA_PLAN:...>>`）
3. Bridge 顯示計畫 + 確認按鈕
4. 「開始執行」→ 依拓撲排序派工（無依賴的並行、有依賴的等前置完成）
5. 全部完成 → 主 agent 最終整合

Plan 也能被 heuristic 自動偵測——`/moa 做一個 hello world` 開頭是動詞「做/建立/實作…」會自動走 plan mode。

---

### Phase 4：Debate 辯論模式（進行中）

跟 blind review 不同，debate 讓兩方**看到對方觀點後互相反駁**：

- Round 1：盲答
- Round 2：看到對方回覆後反駁/修正
- 嚴格 2 方（>2 用 blind review）、固定輪數（不做自動收斂判斷）

觸發：`/moa:debate <問題>`

---

### 配置檔位置

- Preset 定義：`${MEMORY_DIR}/config/moa-presets.json`
- Specialist domain：`${MEMORY_DIR}/config/specialist-domains.json`（`moa-ref-*` 區塊）
- Reference preamble：`specialists/moa-ref-*/preamble.md`
