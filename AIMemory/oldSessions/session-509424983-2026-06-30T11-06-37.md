## Session 歸檔/恢復機制說明

### 核心問題

原本 bridge 的 session 關閉後只留下兩樣東西：
1. **Transcript MD** — 純 markdown，給人看的（不能程式化恢復）
2. **Working State** — 只有任務層摘要（task/next_action），沒有對話上下文

結果是：新 session 建立時，agent 只知道「上次在做什麼任務」，但不知道「對話到哪了、聊了什麼、goal 進度如何」。

### 解法：結構化 JSON 歸檔

```
Session 關閉                          新 Session 建立
     │                                      │
     ▼                                      ▼
exportSession() ──→ saveArchive()     loadArchive() ──→ buildRestorationBlock()
     │                   │                  │                      │
     │                   ▼                  │                      ▼
     │        session-archive-{chatId}.json  │           注入 preamble 尾段
     │                                      │                      │
     ▼                                      └──→ deleteArchive() ──┘
  同時照舊：                                      (防止下次重複注入)
  - saveTranscript (MD)
  - writeWorkingState
  - extractFromSession (抽 facts)
```

### 觸發時機

| 場景 | closeReason | 觸發路徑 |
|------|-------------|---------|
| `/reset` | `reset` | `misc.ts` → `drop("reset")` |
| Idle 30 分鐘 | `idle` | `sweepIdle()` |
| `/restart` / bridge 關機 | `restart` | `shutdown()` |
| Agent process 死掉 | `crash` | `exit` event → `onUnexpectedExit` |
| Health monitor 偵測到死 PID | `health_monitor` | `sweepIdle()` PID probe |

### 與 Working State 的分工

| | Working State | Session Archive |
|--|---------------|-----------------|
| 格式 | Markdown | JSON |
| 內容 | 任務摘要（task/completed/blocked/next_action） | 完整結構（turns/goal/stats/recentSummary） |
| 來源 | agent 回覆中的 `[WS]` token | 程式自動從 ChatSession 提取 |
| 注入位置 | preamble | preamble（在 working-state 之後） |
| 特色 | 精確的 next_action | 對話上下文 + goal 進度 |

兩者互補 — working-state 告訴 agent「下一步做什麼」，archive 告訴 agent「上下文在哪」。

### 設計決策

- **Per-chatId 單檔覆寫**：因為只需要最近一次 session（排除 append-only 因為歷史 session 有 transcript MD）
- **Turn text 截斷 2000 字**：避免歸檔膨脹，完整內容在 transcript
- **恢復只注入摘要**：不灌完整 history 進 preamble（排除全量注入因為會吃掉太多 context budget）
- **注入後自動刪除**：防止下次 session 重複注入相同歸檔

**問題 1：不想帶入上次摘要，要怎麼結束？**

---

**問題 2：一直帶入下個對話有風險嗎？**

風險很低，原因：

| 顧慮 | 實際情況 |
|------|---------|
| Context 膨脹 | restoration block 只有 ~300 字（5 行摘要），佔 preamble 不到 2% |
| 過時資訊誤導 | 只保留「最近一次」session，且只有最後 2-5 句 assistant 摘要（150 字/句截斷） |
