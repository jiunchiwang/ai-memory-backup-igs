## 懸賞令（Wanted Poster）運作機制

### 概念

懸賞令是一種**符號倍率加成系統**。遊戲會指定某些普通符號（M1~M9）為「被懸賞」的對象，當這些符號在盤面上被消除時，觸發懸賞令演出並套用倍率。

### 資料結構

每筆懸賞令（`IBountyData`）包含：
- `BaseSymbol` — 被懸賞的原始符號（M1~M9）
- `BountySymbol` — 對應的懸賞令符號（用於場上 Spine 動畫的 skin）
- `MultipleIndex` — 倍率索引
- `Multiple` — 實際倍率值（如 2x、4x、256x）

### 觸發流程（WantedPosterState）

在每次消除判定後（EXPLODE 之後），狀態機進入 `WANTED_POSTER` 狀態：

```
EXPLODE → WANTED_POSTER → MATCHING_PATCH_UP
```

**步驟：**

1. **判定觸發** — 比對本盤 `AwardDataVec`（被消除的符號）與 `BountyDataVec`（當前懸賞令清單），找出哪些懸賞令的 `BaseSymbol` 有被消除
2. **分組** — 同 `BaseSymbol` 的懸賞令歸為一組，組間依序執行（組內合併）
3. **每組演出**（三階段 pipeline）：
   - **① Mark** — 常駐 UI 上對應的懸賞令 item 播 `Mark` 動畫（高亮提示），完成後接 `Mark_Idle` loop
   - **② Remove ∥ Symbol_In**（並行）— 盤面上被消除位置播爆炸特效 + 場上 spawn 懸賞令 Spine 播 `Symbol_In`（飛入動畫）
   - **③ Mark_Active ∥ Symbol_Active**（並行）— 常駐 item 播 `Mark_Active`（完成後 `active=false` 消失）+ 場上 Spine 播 `Symbol_Active`（激活動畫後回收）
4. **升級倍率底座** — 演出完成後，對消除位置的 Multiplier Base 播升級動畫（`SET_MULTIPLIER_WITH_ANIM`）
5. **無觸發** → 直接跳 `MATCHING_PATCH_UP`

### 常駐 UI（WantedPosterManager）

畫面側邊有一排常駐懸賞令 UI（最多 N 個 `WantedPosterItem`）：

- **資料更新**（`SET_WANTED_POSTER_DATA`）— 每盤開始時 server 下發新的懸賞令清單，Manager 做前後比對：
  - 新出現（`Multiple > 0` 且之前沒有）→ 播 `In` 動畫顯示
  - 消失（`Multiple ≤ 0`）→ 播 `Out` 動畫後隱藏
  - `BaseSymbol` 不變 → 僅更新倍率數字
  - `BaseSymbol` 變了 → 先 `Out` 再重設資料 + `In`
- **響應式排版** — 支援直/橫向 Layout 切換

### 單個 Item（WantedPosterItem）

- Spine 動畫元件，skin 對應 M1~M9
- 倍率 Label 由 `NewBoneFollow` 跟隨 Spine bone 位置
- 動畫集：`In`（出現）、`Out`（消失）、`Mark`（高亮）、`Mark_Idle`（loop 等待）、`Mark_Active`（激活消失）、`Symbol_In`（場上飛入）、`Symbol_Active`（場上激活）

### 倍率計算

同組多筆懸賞令的 `Multiple` 會**連乘**（不是相加），結果顯示在場上 Spine 的倍率 Label 上（純表演用，實際贏分由 server 計算）。

### 與其他系統的交互

- **EffectPlate** — 負責場上 Spine 的 spawn/播放/回收
- **MultiplierBaseManager** — 懸賞令觸發後升級對應位置的倍率底座
- **消除位置過濾** — Wild 位置會被排除（Wild 不算真正的 BaseSymbol 消除）
