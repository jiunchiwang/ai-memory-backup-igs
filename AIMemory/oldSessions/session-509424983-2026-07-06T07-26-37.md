### 🤖 Assistant (下午3:22:28)

Trailer 是 harness 模板在 session 啟動時定格的，沒有「填正確 model」的原生設定。可行做法三種：

**1. CLAUDE.md 加規則（推薦）**
在專案 CLAUDE.md 加一條：「commit 時 Co-Authored-By 的 model 名以 session 環境宣告的實際 model（`You are powered by...`）為準，不照預設模板寫」。agent 自己知道正在跑什麼 model，指示優先於模板。文字層自律，但 CLAUDE.md 指令 Claude 遵循度高。

**2. 直接關掉 attribution（最乾淨）**
`settings.json` 設 `"includeCoAuthoredBy": false` — trailer 整個不寫，沒有錯誤資訊，代價是也沒有 model 紀錄。

**3. git prepare-commit-msg hook（機械強制）**
hook 改寫 trailer，model 名從 `ACP_MODEL` 環境變數讀（bridge spawn 時會傳下去）。最可靠但最工程，且 Claude Code 手動 session 沒這個 env var，要另做 fallback。

我的建議：1（想保留正確紀錄）或 2（覺得 trailer 沒價值）。3 對這個需求過重。
