## Open Knowledge Format (OKF) v0.1 — Google Cloud 開源知識格式

**Repo**: `GoogleCloudPlatform/knowledge-catalog` → `okf/` 子目錄
**Stars**: 4.1k

### 是什麼

OKF 是 Google 提出的一套**通用、vendor-neutral 的知識表示格式**——用 **Markdown + YAML frontmatter** 的純檔案目錄結構來表達「知識」（metadata、context、curated insight）。

核心主張：知識本身就是一堆 `.md` 檔，不需要專屬 SDK、API、schema registry。

### 設計哲學

| 特性 | 說明 |
|------|------|
| Human & Agent readable | `cat` 一下就能讀，LLM 直接當 context 吃 |
| Version-controllable | 放 git、line-by-line diff、PR review |
| Portable | 就是一個目錄，tarball / git / mount 隨便搬 |
| 結構化 + 非結構化並存 | frontmatter 放可查詢欄位，body 放 prose/schema/examples |
| Extensible | 必填欄位極少（只有 `type`），其餘隨便加 |

### Bundle 結構

```
bundle/
├── index.md          # 目錄列表（progressive disclosure）
├── log.md            # 變更歷史
├── datasets/
│   ├── index.md
│   └── sales.md      # 一個 concept = 一個 .md 檔
└── tables/
    ├── orders.md
    └── customers.md
```

### Concept 文件格式

```yaml
---
type: BigQuery Table        # ← 唯一必填
title: Customer Orders
description: 一句話摘要
resource: https://...       # 指向實體資產的 URI
tags: [sales, orders]
timestamp: 2026-05-28T14:30:00Z
---
# Schema
| Column | Type | ... |

# Citations
[1] [來源](url)
```

### 附帶的 Enrichment Agent

用 Google ADK + Gemini 做的 PoC agent，示範「自動產 OKF bundle」：
1. **BQ pass** — 從 BigQuery metadata 自動產出每個 table/dataset 的 concept 文件
2. **Web pass** — LLM 當爬蟲，跟著 seed URLs 抓文件來豐富既有 concept

附帶 3 個範例 bundle（GA4 / StackOverflow / Bitcoin）和 `viz.html` 視覺化（Cytoscape.js 力導向圖 + markdown render）。

### 跟你的 bridge 系統的關聯

OKF 的設計理念跟你目前的 AIMemory 體系（facts → topic shards → wiki pages）高度相似：

| OKF | 你的系統 |
|-----|---------|
| Bundle = 目錄 | `AIMemory/wiki/` |
| Concept = .md + frontmatter | wiki page（concepts/references/lessons/queries） |
| Cross-link = markdown link | wiki 內互連 |
| index.md = progressive disclosure | topic index in preamble |
| log.md = 變更歷史 | hit-log.jsonl / forget-log.md |
| Enrichment Agent（auto-generate） | `/wikisync`（facts → wiki pages） |

主要差異：OKF 多了 **正式的 YAML frontmatter schema**（`type` 必填 + `resource` / `tags` / `timestamp`），以及 **version 概念**。你的 wiki 目前是純 markdown body 沒有 frontmatter。

如果要讓你的 wiki 體系更可互通或被其他 agent/工具消費（例如丟給 Obsidian 當 vault、或跟別的 agent 交換知識），加 OKF frontmatter 是成本很低的升級。
