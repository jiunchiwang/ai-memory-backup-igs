### 🤖 Assistant (下午7:14:15)

這次 upstream 更新做了兩件事：

## 1. 新增 `toAcpMcpServers()` 正規化函式

**問題**：ACP SDK 的 `session/new` 對 MCP server 結構做嚴格 schema 驗證（zod `zMcpServer`）——要求 stdio 類型的 `args` 和 `env` 都存在，且 `env` 必須是 `[{name, value}]` 陣列格式，不是我們設定檔慣用的 `{KEY: "val"}` 物件。HTTP/SSE 的 `headers` 同理。不符合就噴 `-32602`。

**修正**：在 spawn specialist 前統一把設定檔的 MCP server 物件轉成 ACP 合規格式：
- `env: {KEY: val}` → `env: [{name, value}]`
- `headers: {KEY: val}` → `headers: [{name, value}]`
- `disabled: true` 的 server 直接過濾掉
- `args` 缺失時補空陣列

## 2. `TG_ACTIVE_USER_ID` 傳遞給 specialist

**問題**：specialist spawn 的子 process 裡沒帶 `TG_ACTIVE_USER_ID`，導致 specialist 的 memory MCP server 無法解析當前使用者（`list_facts` 會報 "active user id not configured"）。

**修正**：`spawnNew()` 和 `spawnOrReuse()` 的簽名新增 optional `userId` 參數，spawn 時注入 `TG_ACTIVE_USER_ID` 到子 process env。呼叫端（`dispatchToSpecialist`、`dispatchParallel`、`promptProxy`）都已傳入。

---

兩個都是 bug fix，不影響外部 API，合併後不需額外改設定。
