即 **兩個條件任一不滿足**就是「未就緒」：

1. **`embedder === null`** — 模型還沒載完（`pipeline("feature-extraction", MODEL_NAME)` 是 async，首次要下載 ~90MB ONNX 或從 cache 載入）

而 `initEmbedRouter(routing)` 要被呼叫才會啟動，它需要一份有效的 `RoutingV2` JSON。讓我確認呼叫端的條件：原因找到了。Embedding Router 未就緒的可能原因有 **3 個**（依啟動 log 會印對應訊息）：

| 原因 | 啟動 log | 解法 |
|------|---------|------|
| **`notebooklm-routing.json` 不存在** | `skipped: notebooklm-routing.json not found` | 跑 `/refresh-routing` 讓 agent 產出這個檔案（需掛 `notebooklm` MCP） |
| **routing.json 不是 v2 格式** | `skipped: routing.json is not v2` | 同上，重新產出會是 v2 |
| **模型載入或 embed 過程拋錯** | `init failed: <error>` | 通常是 ONNX runtime 問題（記憶體不足、node_modules cache 損壞） |

但注意：**Embedding Router 未就緒不影響 bridge 正常運作**——它只是讓 NotebookLM 語意路由功能失效，其餘 memory recall、skill routing、wiki retrieval 等子系統各自獨立初始化（看 `memory-recall.ts`、`skill-router.ts` 等），不依賴這個 router。
