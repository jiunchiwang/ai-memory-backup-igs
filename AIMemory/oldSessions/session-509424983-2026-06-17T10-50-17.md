2. **blend mode 切換**（`batcher.setBlendMode()` → flush）
3. **clipping attachment** 出現（會呼叫 `clipper.clipStart`，後續 slot 走 clip path 但 texture 仍是判斷依據）

texture 來自 `region.region.renderObject.texture`（即 atlas page 的 GLTexture 物件）。在我的 dummy 環境中，每個 atlas page 會得到一個獨立的 dummy object。所以 **texture identity（page object）+ blend mode** 就是精確 DC 的判定。現在有個問題：dummy texture 目前所有 atlas page 回傳同一個 object `{ setFilters(){}, ... }`，這樣 texture identity 全部相同，DC 永遠是 1。需要讓每個 atlas page 回傳**獨立物件**：完成。精確版 DC 計算邏輯：

1. 遍歷 `skeleton.drawOrder`，跳過 inactive bone 和 ClippingAttachment
2. 取每個 slot 的 `att.region.renderObject.texture`（實際 atlas page 物件）
3. **blend mode 切換** → 斷批（DC++）
4. **texture identity 切換** → 斷批（DC++）
5. 最後一批結束 → DC++

這跟 spine-webgl `SkeletonRenderer.draw` + `PolygonBatcher` 的真實邏輯一致。每個 atlas page 現在會得到獨立的 dummy 物件，所以 `texture !== lastTexture` 的比對能正確觸發。
