# Pattern: MAX WIN（最大獎上限）

## 識別條件

規格書出現以下描述時匹配：
- 「MAX WIN」「最大獎」「贏分上限」「封頂」
- 「贏分達上限時跳最大獎提示」
- 「達上限強制結束免費遊戲 / 獎金遊戲」
- 「報獎分數只給剩餘差值」

這是**橫切機制**（cross-cutting），不是某個 Feature 的子流程；幾乎任何含 FG / 連續獎金累積的機台都可能有。

## 參考實作

| 專案 | 機制 | 特殊點 |
|------|------|--------|
| Clash of Olympus（規格） | MAX WIN | 達上限跳提示、強制結束 FG、報獎只給差值 |

### 核心檔案（規劃中）
- `Manager/MaxWinManager.ts` 或併入結算流程 — 每手結算前檢查累計贏分 + 本手贏分是否超上限
- `GameState/RoundEndState.ts` / `AwardState.ts` → 達標時切入 MAX WIN 提示 + 強制結束分支

## State 映射

**不需要獨立 State**，嵌在結算/報獎階段檢查：

```
AwardState（本手贏分算出後）
  → 檢查 (當前 WIN 欄分數 + 本手贏分) >= MaxWin?
     → 是：報獎分數 = MaxWin − 當前 WIN 分數 → 跳 MAX WIN 提示面板
            → 強制結束 FG / 獎金遊戲（無視剩餘手數）→ 結算面板
     → 否：正常結算 → 下一手
```

## Data 需求

```typescript
interface MaxWinData {
  /** 上限值（server info / 設定提供，通常為押注的固定倍率） */
  maxWinValue: number;
  /** 當前已累計贏分（WIN 欄） */
  currentWin: number;
  /** 本手是否觸發上限（server 判定為準） */
  isMaxWinTriggered: boolean;
}
```

報獎差值公式（規格明定）：
```
報獎分數 = MAX WIN 分數 − 目前 WIN 分欄位分數
```

## 演出時序

| Step | 動作 | 等待方式 | 依賴 |
|------|------|---------|------|
| 1 | 本手所有贏分（收集/線獎/聚寶盆等）算出 | — | Award 完成 |
| 2 | 判斷 (currentWin + 本手贏分) ≥ MaxWin | 即時 | maxWinValue |
| 3 | 達標 → 本手報獎只給差值（封頂） | await count | 差值公式 |
| 4 | 跳 MAX WIN 最大獎提示面板 | await panel.close() | — |
| 5 | 強制結束 FG / 獎金遊戲（無視剩餘旋轉手數） | — | — |
| 6 | 接續結算面板，直接結算該局 | await | — |

## 常見變體

| 變體 | 差異 | 參考 |
|------|------|------|
| 達標封頂給差值 | 報獎只給 MaxWin − currentWin | Clash of Olympus |
| 達標全額給付 | 不切差值，照給但仍強制結束（依設定） | 其他機台 |
| 僅 FG 適用 vs 全場適用 | 上限只在 FG 算 vs MG+FG 都算 | 依遊戲 |
| 提示後是否續播 | 多數強制結束；少數僅提示不結束 | 依設定 |

## 邊界案例

1. **差值計算時機**：必須在「本手所有贏分加總後」才比對上限，不可逐項提前封頂
2. **無視剩餘手數**：FG 達上限後，剩餘免費手數作廢，直接跳結算（規格明示）
3. **跨來源贏分都計入**：收集、線獎、聚寶盆收集等所有類型贏分都算進上限判斷（規格明示）
4. **server 為判定權威**：client 不可自行用本地累計判定封頂，須以 server 的 isMaxWinTriggered / 差值為準（避免浮點/累加誤差）
5. **與乘倍/VS 交互**：乘倍、VS 勝出倍率作用後的最終金額才拿去比對上限
6. **斷線重連**：若上一手已觸發 MAX WIN 結束，重連需直接呈現結算後狀態，不重播提示
7. **MG 單手達標**：若 MG 單手贏分即超上限（罕見），同樣走封頂 + 提示流程

## 常見錯誤

1. **❌ 逐項提前封頂**：差值計算必須在「本手所有贏分加總後」才做；若逐項（Collect 一次、線獎一次）各自封頂 = 總金額比實際 MAX WIN 少
2. **❌ Client 自行判定封頂**：MAX WIN 判定權威是 server（isMaxWinTriggered + 差值由 server 算），client 本地加法有浮點累加誤差，不可自行 compare
3. **❌ 達標後沒強制結束 FG**：規格明示達上限無視剩餘手數直接結算；忘了 = FG 繼續跑但贏分不再增加（UI 混亂）
4. **❌ 不含 Collect 贏分就比對上限**：所有類型贏分（線獎 + Collect + 聚寶盆）都計入上限判斷；漏了 Collect = 可能超額給付
5. **❌ 重連時重播 MAX WIN 提示面板**：若上手已觸發結束，重連後應直接呈現結算後狀態——重播面板 = 玩家誤以為又贏了一次
