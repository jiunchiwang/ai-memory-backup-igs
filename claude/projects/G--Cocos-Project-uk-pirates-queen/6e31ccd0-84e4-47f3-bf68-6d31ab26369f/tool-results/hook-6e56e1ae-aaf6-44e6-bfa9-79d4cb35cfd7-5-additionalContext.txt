**#S777** Commit bugfix for WantedPoster skeleton skin rendering issue in uk_pirates_queen Cocos project (Jun 11, 4:03 PM)

**Investigated**: Reviewed git diff showing changes to WantedPosterManager.ts where three methods (ShowFirstBounty, PlayBounties callback, and EnterAt) had their operation order modified. Examined the root cause where BaseSymbol 9 was incorrectly displaying as 1 on the first hand but correctly on subsequent hands.

**Learned**: Cocos Creator's spine skeleton system has a critical gotcha: calling setSkin() on an inactive node doesn't rebuild render data for already-rendered skeletons, causing old skin graphics to persist. The correct pattern is to activate the node first (node.active = true), then call setSkin/SetWantedPosterData, ensuring the skeleton's render data is properly rebuilt with the new skin.

**Completed**: Fixed skeleton skin rendering bug in WantedPosterManager.ts by reordering operations in three methods to activate nodes before setting skin data. Committed changes with detailed explanation (SHA 00dacb3, 1 file changed, 6 insertions, 3 deletions). Verified fix with device logs and screenshots confirming M9+M7 display correctly. Commit message amended to remove formatting artifacts.

**Next Steps**: Changes are committed to main branch but not pushed. Awaiting decision on whether to push, and whether to record the Cocos skeleton rendering gotcha as a reusable pattern for future reference.
