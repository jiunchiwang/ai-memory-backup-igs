Examined uk-slot-extrabet skill documentation for integration patterns.

**Learned**: Identified two critical non-obvious integration traps when linking ExtraBet to persistent visuals: (1) ForceSetExtraBet's callback receives triggerCb(isActive, isForce=true) without type parameter, breaking type-dependent visual restoration in unshow/replay paths; (2) GameState per-spin resets (like SpinState.OnEnter clearing multiplier boards with SET_MULTIPLIER -1,0) repeatedly erase ExtraBet-activated visuals unless reset logic becomes ExtraBet-aware. Implementation requires non-contiguous type→index mapping [0,1,2,6], centralized ApplyExtraBetMultiplierBase method, and GameView member variable to bridge type information across callback paths.
