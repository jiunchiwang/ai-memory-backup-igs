**#S814** Configure FeatureWheelResult background sprite reference and discover MCP tool limitation (Jun 16, 8:10 PM)

**Investigated**: Attempted to wire m_resultBgSpriteChange component reference using cocos362-mcp-server's component_set_property tool. Tested setting reference via {refNodeUuid} object and null value. Examined prefab serialization format to verify proper Cocos internal reference structure (__id__ objects vs strings). Reviewed project MCP server architecture and tool capabilities.

**Learned**: The cocos362-mcp-server's component_set_property tool has fundamental limitation: it stringifies all values regardless of input type, making it impossible to set component-type references. Tool returns ok:true (false positive) even when it corrupts data by storing literal strings instead of proper Cocos references. Correct serialization format for component references is {"__id__": N}, not string values. Prefab editing requires entering prefab-edit mode in Cocos Editor first.

**Completed**: Manually corrected Node_FeatureWheel.prefab to change m_resultBgSpriteChange from string "null" to proper JSON null value. Documented the MCP bridge limitation in project memory (project_cocos362_mcp.md) with testing date, failure modes, workarounds, and verification procedures. FeatureWheelManager code logic for dual BG+Symbol SpriteChange switching was completed in prior work.

**Next Steps**: User needs to manually wire the component reference in Cocos Editor Inspector by dragging the BG SpriteChange component to the "結果展示牌背景 SpriteChange" field, then save.
